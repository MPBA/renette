Short Description

In the Network Inference procedure many network inference algorithm are implemented, such as correlation-based measures, ARACNE or CLR.
The inference method is computed for all possible pairs of columns in the input file.

Moreover, in case of two networks, they can be compared through the Network Distance  procedure.

Step 2.

Methods selection and parameters set

The network inference algorithm can be chosen among 11 different methods.

  • Select the inference algorithm in the Method drop-down menu

    • cor
    • WGCNA
    • WGCNAFDR
    • bicor
    • bicorFDR
    • TOM
    • ARACNE
    • CLR
    • MINE
    • MINEFDR
    • DTWMIC

    • cor: Pearson correlation coefficient
    WGCNA: Weighted Gene Correlation Network Analysis. the co-expression similarity is defined as a function of the absolute correlation, with soft-thresholding, i.e., the similarity is defined as a power of the absolute correlation [1].
    WGCNAFDR: the WGCNA with False Discovery Rate correction. The FDR controlled procedure defines a rule for choosing which edges to trust (thus to keep) during the network reconstruction phase. We essentially count how many times the absolute correlation level between two nodes is lesser than the correlation between a shuffled version of the sample data. Precisely, the data are shuffled as many times as possible (depending on the sample size) or at least as many times as needed to reach a chosen p-value threshold (see FDR value in the parameter list).
    bicor: the biweight midcorrelation, which is more robust to outliers than the Pearson correlation [1].
    bicorFDR: the bicor similarity score with False Discovery Rate correction.[5]
    TOM: The Topological Overlap Measure (TOM) replaces the original co-expression similarity with a measure of interconnectedness (between couple of nodes) based on shared neighbors. It can be seen as a filter for cutting away weak connections, thus leading to more robust networks than WGCNA [1].
    ARACNE: The Algorithm for the Reconstruction of Accurate Cellular Networks (ARACNE) is an approach relying on the mutual information, which was originally developed for inferring regulatory networks of mammalian cells. It starts with a graph where each pair of nodes are connected if their association is above a chosen threshold. In order to avoid the false positive problem, usually affecting co-expression networks, the Data Processing Inequality (DPI) is then applied for removing the weakest edge of each triplet, thus removing the majority of indirected links. [2]
    CLR: The Context Likelihood of Relatedness (CLR) approach scores the interactions by using the mutual information between the corresponding gene expression levels, coupled with an adaptive background correction step. Although suboptimal if the number of nodes is much larger than the number of variables, it was observed that CLR performs well in terms of prediction accuracy and some CLR predictions in literature were later experimentally validated. [3]
    MINE: one of the Maximal Information-based Nonparametric Exploration (MINE) statistics. [4]
    MINEFDR: the MINE similarity with False Discovery Rate correction. [5]
    DTWMIC: linear combination of Dynamic Time Warping based distance and MIC (i.e., the Maximal Information Coefficient belonging to the MINE family).[6]
  • For each algorithm, the user can set the corresponging parameters:
    • P
    (WGCNA and WGCNAFDR) soft thresholding parameter: the exponent of the similarity function defining the existence of a link. Default value set to 6.
    FDR
    (WGCNAFDR, bicorFDR, MINEFDR) p-value throshold for the FDR procedure. Default value set to 1e-3.
    alpha
    (MINE and MINEFDR) Default value set to 0.6.
    C
    (MINE and MINEFDR) Default value set to 15.
    Measure
    (MINE and MINEFDR) Statistic to be used among the MINE family. Possible values are MIC, MAS, MCN, MEV, and MICR2 (which means MIC-cor^2).

    The Result page

    Process Status

    The result page contains all the information on the submitted job. It includes the job UUID, the submitted procedure, the selected parameters of the running job, the submission time stamp, the status of the process (success, started, failure, running, pending), see faq section for more details on process statuses. Then in the same box, if the job has successfully complete, a link to a zip file with all the results is provided.

    The session will last 7 days: within 7 days after the submission date results and job details are available in the process list. After 7 days the session expires; job details and results are no longer available.

    For each input file a weighted adjacency matrix is returned. In particular the weight is the inference measure between two variable in the input file. 

    For each adjacency matrix it is possible to visualize the network, in which the node dimension is proportional to the node degree and the edge width is proportional to the edge weight given by the adjacency matrix. Node colors denote different communities found applying the Spinglass clustering algorithm on the weighted network (this is only for visualization purpose).
    Networks can be exported as adjacency matrix as "csv" files or as "gml" file format (see here for more details). If column names are provided in the input file they will be used as node identifiers.

    The implementation in based on the R nettools packages available on github
